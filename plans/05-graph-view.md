# План 5. Граф связей заметок (graph view)

Статус: проект (не начат)
Приоритет: средний — фича видимости знаний; ядро (индекс ссылок) уже
готово, работа в основном фронтендная.
Зависимости: механизм настроек плагинов (реализован: `acl.PluginState`,
`pluginsdk.SettingSpec`, admin API `PUT /api/admin/plugins/:id`).

## 1. Цель и мотивация

Obsidian-пользователи ждут граф: визуальную карту заметок и wiki-ссылок.
У платформы уже есть все данные — `packages/links` держит в памяти
полный индекс исходящих ссылок и бэклинков и обновляется на каждый
reindex/save/delete. Не хватает только API-выгрузки и рендера.

Решение из обсуждения (2026-07-19): глобальный граф как отключаемый
UI-плагин `graph`, рендер библиотекой `force-graph` (canvas, d3-force
внутри), фильтрация по ACL на сервере. Локальный граф на странице
заметки — второй этап того же API (`?root=&depth=`).

## 2. Что уже есть

| Компонент | Файл | Использование |
| --- | --- | --- |
| Индекс ссылок (out + backlinks) | `packages/links/index.go` | источник рёбер |
| Метаданные заметок (title, folder) | `packages/search`/`core.NoteService` | узлы |
| ACL-фильтрация списков (`allowRead`) | `packages/api` (tree/search/recent) | образец фильтрации графа |
| UI-плагины с toggle | `plugins.UIPlugin`, `/api/plugins` | регистрация `graph` |
| WS-события vault | `useVaultEvents.ts` | инвалидация графа |

## 3. Дизайн

### 3.1 API

`GET /api/graph` (permission `notes:read`), опционально
`?root=<path>&depth=<n>` для локального графа.

```json
{
  "nodes": [{ "path": "Inbox/Idea.md", "title": "Idea", "folder": "Inbox", "links": 3 }],
  "edges": [{ "from": "Inbox/Idea.md", "to": "Projects/App.md" }]
}
```

Правила:
- узлы и рёбра фильтруются через ACL: узел виден только при
  `allowRead(path)`; ребро — только если читаемы **оба** конца
  (иначе утечка существования скрытой заметки);
- битые ссылки (цель не существует) — по флагу `?ghosts=1` отдаются
  как узлы `{ "ghost": true }`; по умолчанию опускаются;
- размер: карта строится из индекса в памяти за один проход,
  O(notes + links); для типичных хранилищ (≤10k заметок) — миллисекунды,
  кеширование не нужно.

Ядро: метод `NoteService.Graph()` (или экспорт из `links.Index` +
обогащение заголовками), тонкий хендлер `handleGraph`, маршрут со своим
префиксом (не под `/api/note/*path` — gin-гочта из CLAUDE.md).

### 3.2 Плагин

- `plugins.UIPlugin{ID: "graph", …}` — переключается в табе «Плагины»,
  фронтенд прячет пункт меню/страницу при выключенном плагине;
- эндпоинт `/api/graph` при выключенном плагине отвечает 404 (проверка
  `pluginEnabled("graph")` в хендлере, как у templates в
  `handleCreateNote`);
- задел: настройка плагина `maxNodes` (лимит для очень больших vault)
  через готовый механизм `SettingSpec`.

### 3.3 Frontend

- Библиотека: `force-graph` (~40 kB gzip, canvas, zoom/pan/drag из
  коробки, тянет тысячи узлов). Альтернатива без зависимости —
  самописный d3-force, но это ~2-3 дня работы против ~0.5;
  страница графа грузится lazy (`import()`), чтобы не раздувать
  основной бандл (он уже >500 kB — предупреждение vite).
- Страница `/graph`: клик по узлу → `/n/<path>`; hover — подсветка
  соседей; поиск по названию; фильтр по папке; toggle «сироты».
- Цвет/размер узла: по папке верхнего уровня / количеству связей.
- Тема: перерисовка цветов при смене light/dark (store `theme`).
- Инвалидация: `useVaultEvents` уже слушает WS — добавить
  `invalidateQueries(['graph'])` на note created/deleted/renamed;
  debounce, чтобы массовые reindex не дёргали перерисовку.
- Локальный граф (этап 2): блок на странице заметки, `?root=&depth=2`.

## 4. Этапы

1. **API + ядро** (0.5 дня): `NoteService.Graph()`, хендлер с
   ACL-фильтрацией, маршрут, тесты (ACL: скрытый узел не светится ни
   узлом, ни ребром).
2. **UI-плагин + страница** (1 день): регистрация `graph`, lazy-страница
   с force-graph, клик/hover/поиск, i18n (en+ru), пункт в сайдбаре.
3. **Live-обновление + фильтры** (0.5 дня): WS-инвалидация, фильтр
   папок, сироты, цвета по папкам.
4. **Локальный граф на странице заметки** (0.5 дня): `?root=&depth=`,
   компактный блок под бэклинками.

## 5. Риски

- **Очень большие vault** (>20k узлов): canvas-рендер деградирует —
  лимит `maxNodes` + предупреждение, WebGL (sigma.js) только если
  реально понадобится;
- **ACL-утечки**: рёбра к скрытым заметкам — покрыть тестом на этапе 1;
- **Бандл**: force-graph обязателен только на странице графа —
  code-split через lazy route.

## 6. Критерии приёмки

- `/graph` показывает все читаемые пользователем заметки и связи;
  пользователь с ACL-ограничением не видит скрытых узлов/рёбер (тест);
- клик открывает заметку; hover подсвечивает соседей;
- выключение плагина в настройках прячет страницу и 404-ит API без
  рестарта;
- создание/удаление заметки обновляет граф без перезагрузки страницы;
- обе локали, светлая/тёмная темы.
