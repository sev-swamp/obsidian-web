# План 6. Roadmap: направления развития

Статус: идеи / приоритизация (2026-07-19)
Назначение: копилка проработанных направлений. Каждое крупное — кандидат
на собственный план (как 03/04/05); мелкие можно брать в работу сразу.

## 1. Где мы сейчас

Ядро зрелое: заметки + wikilinks + рендер (mermaid/mathjax/callouts),
git-история с корзиной 2.0, optimistic locking + presence, поиск,
бэклинки, ACL на папки, роли/группы/SSO, API-токены, плагины с
настройками (templates, vault-stats, recent-changes), i18n en/ru.
Слабые места — UX редактирования, работа с вложениями, «знаниевые»
фичи поверх frontmatter и производительность фронтенда на больших vault.

## 2. Направления

### A. Редактор и UX заметок (высокий приоритет)

| Идея | Что есть | Оценка |
| --- | --- | --- |
| Вставка изображений из буфера/drag-drop прямо в редактор (upload + `![[...]]`) | `/api/upload`, `files:upload` | 1 день |
| Автосохранение черновика (localStorage) — защита от потери при закрытии вкладки | optimistic locking готов | 0.5 дня |
| Live-preview: сплит «код | рендер» или предпросмотр по хоткею | рендер уже на сервере (`note.html`) | 1–2 дня |
| Панель свойств: inline-редактирование frontmatter (type, status, tags…) из режима просмотра | панель уже рендерится, `PUT /api/note` | 1–2 дня |
| Vim-режим / доп. хоткеи как personal prefs | стор prefs готов | 0.5 дня |
| Переименование/перемещение заметки из UI (сейчас только создание/удаление) | history умеет rename-коммиты? проверить | 1–2 дня |

### B. Знаниевые фичи (средний приоритет)

- **Граф связей** — план 05 (готовый дизайн).
- **Браузер тегов**: панель всех тегов с счётчиками, клик = поиск.
  Индекс тегов уже есть в search. ~1 день.
- **Dataview-lite**: кодовый блок ```query``` (фильтр по frontmatter:
  `type=project AND status=active`, сортировка, таблица/список).
  Опирается на поисковый индекс; рендер на клиенте. 2–3 дня, план.
- **Календарь / daily notes**: плагин поверх templates
  (настройки: папка, шаблон дня), мини-календарь в сайдбаре. 1–2 дня.
- **Сохранённые поиски / умные папки** в сайдбаре. 1 день.

### C. Платформа и плагины (средний)

- **UI-слоты для плагинов**: сейчас UI-плагины только скрывают/показывают
  захардкоженные секции. Дать плагинам объявлять панели сайдбара и
  блоки страницы заметки (манифест + lazy-загрузка бандла из
  `/api/plugins/<id>/ui.js`). Это большой шаг к экосистеме — отдельный
  план. 3–5 дней.
- **SSO как внешний плагин** — план 04.
- **Вложения**: страница «Файлы» (список, превью, где используется,
  orphan-cleanup), лимиты размера. 2 дня.
- **Экспорт**: заметка/папка → HTML/PDF; публичная share-ссылка
  (read-only токен на путь — механика API-токенов уже есть). 2–3 дня,
  share-ссылки требуют дизайна безопасности (план).

### D. Производительность и качество (фоновая полоса)

- **Code-splitting фронтенда**: бандл 1.5 MB (+ mermaid-чанк 690 kB);
  lazy-роуты для Settings/Trash, mermaid/mathjax по требованию. 0.5–1 день.
- **Подсветка кода через chroma-классы** вместо invert-фильтра:
  `WithClasses(true)` + две честные темы CSS (сейчас — инверсия
  светлой темы, хрупко: баг с yaml-значениями 2026-07-19). 0.5 дня.
- **Виртуализация дерева/списков** для vault >5k заметок. 1 день.
- **Метрики/health**: /healthz, /metrics (prometheus), время индексации,
  размер индекса. 1 день.
- **E2E-тесты** (playwright): login → создать по шаблону → редактировать
  → конфликт → корзина → восстановить. 2 дня, окупается сразу.

### E. Эксплуатация (низкий, по мере необходимости)

- Бэкапы: периодический push vault-репозитория на remote (настройка
  history), или интеграционный хук. 1 день.
- Политика корзины: авто-purge старше N дней (настройка). 0.5 дня.
- Мульти-vault (несколько корней, переключатель) — большой план,
  трогает ACL/индексы/URL-схему. Только при реальной потребности.

## 3. Рекомендуемый порядок

1. **A: вставка изображений + автосохранение** — самый ощутимый
   ежедневный выигрыш при малой цене.
2. **План 05 (граф)** — эффектная фича, дизайн готов.
3. **D: code-splitting + chroma-классы** — гигиена перед ростом.
4. **B: браузер тегов → dataview-lite** (по нарастающей).
5. **C: UI-слоты плагинов** — когда накопится 2–3 кандидата на
   выносные панели (граф, календарь, теги — уже три).

## 4. Не делаем (осознанно)

- WYSIWYG-редактор «как Notion» — конфликтует с markdown-первичностью
  и Obsidian-совместимостью vault.
- Реалтайм-коллаборация (CRDT) — optimistic locking + presence
  покрывают команду разумного размера; CRDT умножает сложность ядра.
- Собственный формат метаданных — только frontmatter, как в Obsidian.
