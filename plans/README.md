# Планы развития (командная работа)

Проектные документы для передачи в реализацию. Каждый план самодостаточен:
цель, привязка к текущему коду, дизайн, API, этапы, риски, критерии приёмки.

| № | Документ | Суть | Приоритет | Статус |
| - | -------- | ---- | --------- | ------ |
| 1 | [01-safe-collaboration.md](01-safe-collaboration.md) | Безопасная совместная работа: git-история изменений, защита от перезаписи (optimistic locking), присутствие «кто в заметке», корзина, авторство | Высокий | готово |
| 2 | [02-access-control.md](02-access-control.md) | Управление доступом: ACL на папки, группы, управление пользователями из UI, отзыв сессий, API-токены, задел под SSO | Средний | готово |
| 3 | [03-trash-v2.md](03-trash-v2.md) | Корзина 2.0: явный индекс вместо log-scan, честный purge (ACL, (path, rev)), настоящее безвозвратное удаление (переписывание истории), источник восстановления, UX верхней ревизии | Высокий | проект |
| 4 | [04-sso-plugin.md](04-sso-plugin.md) | SSO как внешний плагин: сервисные токены, провижининг с потолком ролей, одноразовый login-code, вынос OIDC из ядра | Средний | проект |
| 5 | [05-graph-view.md](05-graph-view.md) | Граф связей заметок: API из индекса ссылок с ACL-фильтрацией, отключаемый UI-плагин, force-graph, локальный граф заметки | Средний | проект |

Рекомендуемый порядок: план 1 → план 2 (выполнены) → план 3 (этапы 1–4
не зависят от плана 4) → план 4. Этап «переписывание истории» плана 3 —
самый тяжёлый, можно вести последним.
