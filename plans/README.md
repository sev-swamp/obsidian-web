# Планы развития (командная работа)

Проектные документы для передачи в реализацию. Каждый план самодостаточен:
цель, привязка к текущему коду, дизайн, API, этапы, риски, критерии приёмки.

| № | Документ | Суть | Приоритет |
| - | -------- | ---- | --------- |
| 1 | [01-safe-collaboration.md](01-safe-collaboration.md) | Безопасная совместная работа: git-история изменений, защита от перезаписи (optimistic locking), присутствие «кто в заметке», корзина, авторство | Высокий |
| 2 | [02-access-control.md](02-access-control.md) | Управление доступом: ACL на папки, группы, управление пользователями из UI, отзыв сессий, API-токены, задел под SSO | Средний |

Рекомендуемый порядок: план 1 → план 2 (план 1 вводит «actor» в core,
который переиспользуется планом 2; жёсткой блокировки нет).
